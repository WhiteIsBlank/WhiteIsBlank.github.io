先用Die打开文件，发现是ELF64位文件。用IDA64打开。

首先进行字符串查找，发现flag字符，打开该处的反汇编代码。总体观察函数，会发现结尾有一个分支语句，当asc_601060数组中的某一个值等于‘#’，就会出现”Congratulations!”。往上会有一个大的while循环，当v3的数值大于s1数组索引长度时，如果满足v5不为0则结束循环。然后前面有几个分支语句的判断会对v6赋值，之后v6赋值给v5。

main函数在输入s1后，出现了一个if判断语句，当它没有判断成功时，程序才会往下继续执行。

由此处可以知道s1数组的长度为24，且s1数组的前五个字符为”nctf{“,然后数组*(byte_6010BF + 24)的值是‘}’;

继续往下是一个大的while循环;

判断v4的值，分别进行了4种不同运算。

往下分析是一个if判断，如果成立就会直接跳转到LABEL_21，判断错误，退出程序。因此要使得其中函数的返回值为非零值

分析sub_400690函数

返回的结果是result的值，其中的a1是数组asc_601060的地址，a2为v10[0]的值，a3为v9的值

asc_601060数组（由此也可粗略的判断这是一道迷宫问题）：

在sub_400690函数中，result的值是赋值等式的结果取十六进制数的最低位的值，即当result等于‘ ’或是‘#’时返回1，为‘*’时返回0。

即‘#’为出口，‘ ’为通路，‘*’为墙。

与此同时result的第一次赋值时是取asc_601060数组的初始地址，进行了两次的偏移，第一次是a2，第二次是8*a3。由此可以很容易的知道迷宫的长度为8，并且v10[0]为左右向，v9为上下向。

即可得到迷宫：

之后再对之前的四个分支语句进行逐一的分析。

其中sub_400650和sub_400670，sub_400660和sub_400680函数内容相同

sub_400650函数

对输入的数值进行减一的操作（均是对v10和v9相应地址的值进行了改变），当超过迷宫大小范围时则返回0

sub_400660函数

对输入的数值进行加一操作，当超过迷宫大小范围时则返回0

结合之前对v10和v9移动的大致方向的判断，就可以得出当v4 == ‘O’(左)，v4 == ‘o’(右)，v4 == ‘.’(上)，v4 == ‘0’(下)

得出flag为nctf{o0oo00O000oooo…OO}

从打印的一些字符来看，很容易的得知是一道迷宫问题

由while循环的最后两个分支可知，‘1’为墙，‘#’为出口。

其中v7数组的初始位置和v3数组的初始位置刚好相差41位，则可以直接将v7[5 * *(_DWORD )&v3[25] - 41 + v4]看做v3[5v3[25] + v4]，由此可知v3初始位置为迷宫起点，v3[25]为上下向，v4为左右向。与此同时迷宫的长度为5

攻防世界_easy_maze

将程序拖入Die发现是一个64位ELF文件，用IDA64打开，查看main函数。

程序中Step_0函数是给v6数组中未赋值的内容进行赋值的，Step_1是用赋值之后的v6去给v5进行赋值，最终得到迷宫的地图，再将地图传入Step_2

当分析到Step_1函数的时候发现，在函数中有个getAStart函数

进入该函数后又有一个getA函数

进入getA函数后就会一直循环的进入同一个getA函数，导致无法分析程序

于是只能通过动态调试的方法来得到地图，此处使用IDA远程动态调试linux的文件

在Step_1函数处下一个断点，F9运行，进入动态调试界面后，单步F8步过函数，直接到v3，将鼠标箭头放置到v5数组处，即可看见v5数组已经被赋值了，记下赋值的内容

进入Step_2函数，其中出现了(*a1)[7 * v10 + v9] == 1

由此可以看出，v10应该为迷宫的上下向，v9为左右向，并且迷宫每七个字符为一行（7列），由于v10和v9均为0，则迷宫的起点为（0,0）。while循环仅在a1数组取值为1的时候执行，可以知道1为通路，0为墙。在函数的末尾处有一个双分支的判断语句当v10 == 6 && v9 == 6时，则走迷宫成功，可以得知终点为（6,6）。接着分析方向，由于v9为左右向，那么++v9就为右，—v9为左，v10同理，得出方向‘d’右，‘s’下，‘w’上，’a‘左。

将以上的条件代入自动走迷宫脚本

在Step_2处下断点，再次对程序进行动态调试，并且在终端中输入得到的结果，用F8步过函数，即可得到flag

最终得到flag为UNCTF{ssddwdwdddssaasasaaassddddwdds}

总结：①在程序陷入无限循环导致无法自主得出一部分内容的时候，可以选择动态调试，跳过某一过程直接获取数值

​ ②对于得到一大堆数据内容（没有任何的符号），但又需要将其变为数组的时候，可以考略利用python，input循环输入（直接将数据复制粘贴到输入框中），再打印将其变为元组即可。

​ ③对于linux中的ELF文件动态调试除了使用IDA，还可以直接在linux中利用gdb进行调试（需要进一步学习）
